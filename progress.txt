## Codebase Patterns
- In this repository, run both `npm run typecheck` and `npm run lint` before committing feature work.
- If Next.js needs to be initialized in a non-empty repo root, scaffold in a temporary directory and copy files into place.
- Convex schema in this codebase uses snake_case field names and numeric `*_at` timestamps, with explicit `by_*` index naming.
- Without Convex codegen checked in, define functions with `mutationGeneric/queryGeneric` from `convex/server` to keep TypeScript passing.
- Without `convex/_generated/api`, cron jobs can still target functions using `makeFunctionReference` with `"module:function"` names.
- For third-party integrations, keep reusable helpers in `lib/` and expose credential readers for both `process.env` and Convex `ctx.env.get(...)`.
- When a mutation needs async side effects (e.g., outbound messages), schedule an action via `ctx.scheduler.runAfter` and target it with `makeFunctionReference`.
- For Next.js server-rendered pages without Convex codegen, use `ConvexHttpClient` + `makeFunctionReference` to call queries/mutations from server components and server actions.
- For time-based UI permissions (e.g., reschedule windows), compute a backend boolean from `session.date` + `session.time` in Convex queries and let UI render from that flag.
- For modal form flows backed by Next.js server actions, return UX status through URL query params (`status`/`error`) after `redirect(...)` so refreshes stay deterministic.
- For admin auth in this codebase, keep credential verification in a Convex action and consume it from NextAuth Credentials provider (`lib/auth.ts`) so role/username live in JWT session callbacks.
- For admin session participant lists, query attendance via `attendance_records.by_session_id` and compute the latest record per participant before rendering status text.
