## Codebase Patterns
- In this repository, run both `npm run typecheck` and `npm run lint` before committing feature work.
- Convex schema in this codebase uses snake_case field names and numeric `*_at` timestamps, with explicit `by_*` index naming.
- For third-party integrations, keep reusable helpers in `lib/` and expose credential readers for both `process.env` and Convex `ctx.env.get(...)`.
- For externally shared links, resolve hostnames through `lib/appBaseUrl.ts`; `APP_BASE_URL` is required and normalized to an HTTPS origin (no fallback host).
- For admin auth in this codebase, keep credential verification in a Convex action and consume it from NextAuth Credentials provider (`lib/auth.ts`) so role/username live in JWT session callbacks.
- For role-based button visibility, check `session.user.role === 'super_admin'` from NextAuth session on server-side before rendering edit/cancel buttons.
- Cancelled sessions must be excluded from the terms acceptance session selector dropdown by filtering `status !== 'cancelled'`.
- Class cancellation requires validation: check for active future sessions (`status !== 'cancelled' AND date >= today`) before allowing cancellation.
- Keep persistent admin navigation in `app/admin/layout.tsx` and keep route highlighting logic in a client-side `app/admin/admin-nav.tsx` component.
- For local Playwright auth flows with NextAuth JWT sessions, start dev server with a fixed `NEXTAUTH_SECRET` to avoid `JWT_SESSION_ERROR` decryption failures.
- When Convex function signatures change, run `npx convex dev --once` before local browser checks so Next.js and direct test calls both use the updated backend contract.
